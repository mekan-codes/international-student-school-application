Build Version 1 of a full-stack web app called “International Student School Support App”.

Purpose:
This app is for international students at a school. Its long-term goal is to centralize systems like substitute food management, cleaning schedules, borrowing shared resources, and suggestions. However, Version 1 should focus only on the substitute food management module.

Important:
Do not build cleaning, borrowing, or suggestion board yet. Only prepare the project structure so those modules can be added later.

Technology:
Use a simple stack that works well in Replit.
Make the project runnable immediately.
Use a database for persistent data.
Use clean, understandable code.
Add comments where useful.

User roles:
1. Admin
2. Student

Student fields:
- id
- name
- student_id
- email
- password
- is_sub_food_member: true/false
- created_at

Admin features:
- admin login
- admin dashboard
- manage students
- add/edit/delete students
- mark students as substitute food members
- create/edit/delete food items
- add stock to warehouse
- transfer stock from warehouse to locker
- view warehouse inventory
- view locker inventory
- view inventory logs/history
- see low-stock alerts

Student features:
- student login
- student dashboard
- view whether they are a substitute food program member
- view available locker food
- see food name, category, and quantity
- students must NOT be able to edit inventory

Inventory model:
There are two separate storage locations:
1. Warehouse
2. Locker

Warehouse:
- main food storage
- managed only by admin

Locker:
- food currently available to students
- students can view it
- admin updates it

Food item fields:
- id
- name
- category
- low_stock_threshold
- is_active
- created_at

Inventory structure:
Each food item should have:
- warehouse_quantity
- locker_quantity

Inventory rules:
- Admin can add stock to warehouse
- Admin can transfer stock from warehouse to locker
- When transferring stock:
  - warehouse quantity decreases
  - locker quantity increases
- Prevent negative quantities
- Validate all quantity inputs
- Students cannot change quantities
- Every stock change must create a log entry

Inventory log fields:
- log_id
- food_id
- food_name
- action_type
- quantity
- source_location
- destination_location
- performed_by_user_id
- performed_by_user_name
- timestamp
- note

Action types:
- add_to_warehouse
- transfer_to_locker
- adjust_warehouse
- adjust_locker

Low-stock alert:
- If locker_quantity <= low_stock_threshold, show a clear warning on the admin dashboard
- Show low-stock food items in a separate alert section
