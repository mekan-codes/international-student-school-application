4. CLEANING SESSION DATE RANGE AND POSTPONE WORKFLOW
==================================================

Change cleaning sessions from exact time range to date-range based workflow.

Cleaning is usually assigned for a period like:
Friday to Sunday, ending Sunday 23:59.

Update cleaning sessions:
* use start_date
* use end_date
* remove or de-emphasize start_time/end_time from UI
* if existing start_time/end_time fields exist, keep them for compatibility but do not require them
* display sessions as date ranges

Example:
May 8, 2026 – May 10, 2026

Cleaning session status values should support:
* scheduled
* marked_done
* approved
* postponed
* cancelled

Task status values can remain:
* assigned
* marked_done
* verified_done
* missed

Student/team workflow:
* team members mark subtasks done
* when all subtasks are marked_done or verified_done, show that the team submitted/completed the work
* staff must approve the whole cleaning session
* after staff approval, session status becomes approved
* assigned team can see that their work was approved

Admin/manager workflow:
* create cleaning session with assigned team and start_date/end_date
* add subtasks
* postpone session:
  - choose new start_date
  - choose new end_date
  - optionally add postpone note
  - status becomes postponed or returns to scheduled with a visible postponed history/note
* approve session:
  - verify/approve the team’s work
  - session status becomes approved
* cancel session if needed

Postpone behavior:
* If Team 1 cannot finish this week, admin/manager can postpone the same session to next week.
* Preserve the assigned team and tasks.
* Update the date range.
* Show a note/history that it was postponed if easy.

Validation:
* end_date cannot be before start_date
* cancelled sessions cannot be marked done by students
* approved sessions cannot be changed by students
* students can only mark tasks for sessions assigned to their team
