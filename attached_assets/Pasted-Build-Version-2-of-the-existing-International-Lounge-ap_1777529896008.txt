==================================================
FEATURE 1: ANNOUNCEMENTS
==================================================

Purpose:
Admins/managers can post official announcements. Students can read published announcements.

Roles:
Admin:
- create announcements
- edit announcements
- delete announcements
- publish/unpublish announcements
- view all announcements

Manager:
- create announcements
- edit announcements
- delete announcements
- publish/unpublish announcements
- view all announcements

Student:
- view published announcements only
- cannot create/edit/delete announcements
- cannot see unpublished announcements

Announcement fields:
- id
- title
- content
- author_user_id
- author_name
- priority
- target_audience
- is_published
- created_at
- updated_at

Priority values:
- normal
- important
- urgent

Target audience values:
- everyone
- all_students
- sub_food_students
- staff_only

Visibility rules:
- everyone: visible to all logged-in users
- all_students: visible to students only
- sub_food_students: visible only to students with sub-food access
- staff_only: visible only to admins/managers
- students must never see unpublished announcements

UI behavior:
- Date/time should be displayed like modern messenger/post UI:
  - near the title or beside the announcement header
  - NOT inside the message body
- Show author separately.
- Show priority as a badge.
- Show target audience as a badge for admin/manager only.
- Students should not see internal management labels unnecessarily.
- Sort newest first.

Pages/routes:
1. Admin/Manager Announcements Management page
   - list all announcements
   - create announcement form
   - edit announcement
   - delete announcement with confirmation
   - publish/unpublish toggle
   - priority badge
   - target audience badge
   - published/unpublished badge
   - newest first

2. Student Announcements page
   - list only published announcements visible to that student
   - show title, content, priority, author, date/time
   - newest first
   - clean card layout

3. Homepage integration
   - Add a “Recent Announcements” section/card on student Homepage.
   - Show latest 3 visible published announcements.
   - Add “View all” link to Announcements page.
   - For admin/manager Homepage, optionally show recent announcements summary.

Sidebar:
- Replace the current “Announcements Coming Soon” placeholder with a real Announcements link.
- Show it to all logged-in users.
- Admin/manager should open the management view.
- Student should open the read-only student view.

Optional if simple:
Announcement reactions.
- Add simple reactions under announcements: 👍 ❤️ ✅ 👀
- Users can react to an announcement.
- Each user can have only one reaction per announcement.
- Clicking another reaction changes their reaction.
- Clicking same reaction again removes it.
- Show reaction counts under the announcement.
- If this adds too much complexity, leave it for later and mention it in docs as planned.

==================================================
FEATURE 2: REQUESTS TO INTERNATIONAL DEPARTMENT
==================================================

Purpose:
Students can submit requests/questions to the International Department.
Admins/managers can review, respond, and update status.

This is a support-ticket system, not borrowing.
Borrowing physical items will be a separate future module.

Roles:
Student:
- create a request
- view own requests only
- view status and admin/manager response
- cannot view other students’ requests
- cannot change status
- cannot edit admin response

Admin:
- view all requests
- filter requests
- update status
- write/edit response
- mark handled by themselves
- delete request if needed

Manager:
- same operational permissions as admin for requests
- view all requests
- filter requests
- update status
- write/edit response
- mark handled by themselves

Request fields:
- id
- student_user_id
- student_name
- category
- title
- description
- status
- admin_response
- handled_by_user_id
- handled_by_name
- created_at
- updated_at

Categories:
- Food
- Dormitory
- Documents
- School life
- Health
- Other

Status values:
- submitted
- in_review
- resolved
- rejected

Validation:
- category is required
- title is required
- description is required
- status defaults to submitted
- students can only see their own requests
- title and description should not be empty
- response can be empty until staff replies

Pages/routes:
1. Student Requests page
   - create request form
   - list my requests
   - show category
   - show status badge
   - show date/time near the request header, not inside the text body
   - show admin/manager response if available
   - newest first

2. Admin/Manager Requests Management page
   - list all requests
   - filter by status
   - filter by category
   - filter by student if easy
   - search by title/student if easy
   - open request details
   - update status
   - write/edit response
   - delete request with confirmation if needed
   - newest first

Sidebar:
- Replace the current “Requests to International Department Coming Soon” placeholder with a real Requests link.
- Students see “Requests”
- Admin/manager see “Manage Requests” or “Requests”
