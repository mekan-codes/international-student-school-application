FEATURE 1: BORROWING SYSTEM
==================================================

Purpose:
Allow students to request shared physical items such as umbrellas, chargers, adapters, calculators, cleaning tools, board games, or sports equipment.

This is separate from Requests to International Department.
Requests = asking for help/service.
Borrowing = requesting a physical shared item with quantity tracking.

Roles:
Admin:
- create/edit/delete borrowable items
- set total and available quantity
- approve/reject borrow requests
- mark items as returned
- view all borrowing history

Manager:
- same operational borrowing permissions as admin
- create/edit/delete borrowable items
- approve/reject borrow requests
- mark returned
- view all borrowing history

Student:
- view available borrowable items
- submit borrow request
- choose quantity
- choose borrow_until_date
- view own borrowing requests/history
- cannot approve/reject requests
- cannot edit item inventory

Borrowable item fields:
- id
- name
- category
- description
- total_quantity
- available_quantity
- is_active
- created_at
- updated_at

Borrow request fields:
- id
- item_id
- item_name
- student_user_id
- student_name
- quantity
- borrow_until_date
- status
- requested_at
- approved_at
- rejected_at
- returned_at
- handled_by_user_id
- handled_by_name
- staff_note

Status values:
- pending
- approved
- rejected
- returned

Rules:
- Student request form should NOT ask for a reason.
- Student request form should ask for borrow_until_date.
- borrow_until_date is required.
- borrow_until_date cannot be in the past.
- When student submits request, status = pending.
- Submitting request does NOT decrease available_quantity.
- When admin/manager approves, available_quantity decreases.
- Admin/manager cannot approve if available_quantity is insufficient.
- When admin/manager marks returned, available_quantity increases.
- available_quantity must never exceed total_quantity.
- available_quantity must never go below 0.
- Students can only view their own borrow requests.
- Admin/manager can view all requests.
- Admin/manager can add staff_note when rejecting or managing a request.

Pages/routes:
1. Student Borrowing page
   - list active borrowable items
   - show available / total quantity
   - request borrow button/form
   - request quantity
   - borrow_until_date
   - list my borrow requests
   - status badges
   - newest first

2. Admin/Manager Borrowing Management page
   - manage borrowable items
   - add/edit/delete items
   - set total/available quantity
   - list all borrow requests
   - filter by status
   - approve/reject pending requests
   - mark approved requests as returned
   - status badges
   - newest first

FEATURE 2: CLEANING TEAMS AND CLEANING SESSIONS
==================================================

Purpose:
Make cleaning duties easier to organize and follow.

Cleaning should be team-based, not only individual assignment-based.

Concept:
Admin/manager creates cleaning teams.
Each team contains selected students.
Admin/manager creates a cleaning session and assigns one team to it.
Each session has subtasks such as:
- Vacuuming
- Cleaning tables
- Taking out trash
- Organizing food locker

Team members can mark subtasks as done.
Admin/manager verifies the subtasks.
The session becomes completed only when all subtasks are verified_done.

Roles:
Admin:
- create/edit/delete cleaning teams
- add/remove students from teams
- create/edit/delete/cancel cleaning sessions
- assign a team to a cleaning session
- add/edit/delete subtasks
- verify marked-done subtasks
- mark tasks missed
- view all sessions and teams

Manager:
- same operational cleaning permissions as admin
- create/edit/delete cleaning teams
- manage team members
- create/edit/delete/cancel sessions
- verify marked-done subtasks
- mark tasks missed

Student:
- view cleaning sessions assigned to their team
- view subtasks
- mark subtasks as done
- add optional student note when marking done
- cannot verify their own completion
- cannot edit teams or sessions

CleaningTeam fields:
- id
- name
- description
- created_by_user_id
- created_at
- updated_at

CleaningTeamMember fields:
- id
- team_id
- student_user_id
- student_name
- added_at

CleaningSession fields:
- id
- title
- description
- location
- team_id
- team_name
- scheduled_date
- start_time
- end_time
- status
- created_by_user_id
- created_at
- updated_at

CleaningTask fields:
- id
- session_id
- task_name
- status
- student_note
- marked_done_by_user_id
- marked_done_by_name
- marked_done_at
- verified_by_user_id
- verified_by_name
- verified_at
- admin_note

Cleaning session status values:
- scheduled
- completed
- cancelled

Cleaning task status values:
- assigned
- marked_done
- verified_done
- missed

Rules:
- Admin/manager can create a cleaning team with selected students.
- Admin/manager can edit team members later.
- A student can belong to multiple teams if needed.
- Admin/manager can create a cleaning session assigned to one team.
- A session must have at least one cleaning task/subtask.
- A cleaning task starts as assigned.
- Only students in the assigned team can mark that session’s subtasks as done.
- Any team member can mark any subtask for their team as done.
- When a student marks a task done:
  - task status becomes marked_done
  - marked_done_by_user_id/name/time are saved
  - optional student_note is saved
- Admin/manager can verify marked_done tasks:
  - task status becomes verified_done
  - verified_by_user_id/name/time are saved
- Admin/manager can mark a task missed.
- Students cannot verify tasks.
- Students cannot mark tasks for teams they are not in.
- Cancelled sessions should not allow students to mark tasks done.
- If all subtasks in a session are verified_done, automatically set session status to completed.
- If any subtask is not verified_done, session remains scheduled unless cancelled.
- Use clear status badges.

Pages/routes:
1. Admin/Manager Cleaning page
   Sections:
   - Cleaning Teams
     - create team
     - edit team
     - add/remove students
     - show team members
   - Cleaning Sessions
     - create session
     - assign team
     - add subtasks
     - edit/cancel/delete session
     - view session subtasks
     - verify marked_done tasks
     - mark tasks missed
     - filter by date/status if easy

2. Student Cleaning page
   - show sessions assigned to teams the student belongs to
   - show session title, date/time, location, team name, status
   - show subtasks
   - allow mark subtask as done
   - optional student note when marking done
   - show which tasks are verified_done
   - show which tasks are still waiting for verification
   - students only see their own team sessions
